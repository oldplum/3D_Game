# 🎮 打砖块游戏升级说明

## 🚀 新增核心特性

### 1️⃣ **生命值 + 重玩系统**
- **3 条命**：游戏开始时拥有 3 条命
- **掉球机制**：球触及底部 → **-1 命**，而非直接游戏结束
- **命数显示**：左上角实时显示 `Lives: X`（命数 ≤1 时显示红色警告）
- **重开游戏**：游戏结束后，按 **R** 重新开始，或 **ESC** 退出
- **保留分数历史**：每条命失败后球恢复但分数保留

---

### 2️⃣ **关卡难度递进系统**
| 关卡 | 球速度 | 砖块构成 | 难度提示 |
|------|------|---------|---------|
| 1    | 1.0x | 8 个绿普通砖 | 🟢 入门 |
| 2    | 1.3x | 5 绿 + 3 蓝硬砖 | 🟢🔵 稍难 |
| 3    | 1.6x | 4 绿 + 3 蓝 + 1 金稀有 | 🟢🔵🟡 中等 |
| 4+   | +30% | 3 绿 + 3 蓝 + 2 金 | 🟡⭐ 困难 |

**关键机制：**
- 每关球速度 **+30%**（防止过简单）
- 砖块类型逐关升级，提高战术复杂度
- 全砖清空 → 自动进入下一关

---

### 3️⃣ **砖块类型 & 彩色计分系统**

#### 砖块种类
| 砖块 | 颜色 | 耐久 | 基础分 | 说明 |
|------|-----|------|--------|------|
| **普通砖** | 🟢 绿 | 1 | 10 分 | 一击摧毁 |
| **硬砖** | 🔵 蓝 | 2 | 25 分 | 需要击中 2 次，显示耐久数字 |
| **金砖** | 🟡 金 | 3 | 50 分 | 稀有高价值，需要击中 3 次 |

#### 计分倍数（连击系统）
- **基础分数** = 砖块基础分 × (1 + ⌊连击数 / 5⌋)
- 连续击中砖块 → 连击计数递增
- 每 5 连击触发 **+1.0x 倍数**
  - 5 连击：10 分砖 = **10 × 1.2 = 12 分**
  - 10 连击：25 分砖 = **25 × 1.4 = 35 分**
  - **连击中断** → 计数重置为 0

**UI 显示：**
- 右上角显示 `Combo: X x1.3`（当前连击数和倍数）


## 🎯 游戏流程

1. **第 1 关**：击破 8 个绿普通砖
   - 球速：1.0x (2, 2)
   - 推荐策略：学习挡板控制和碰撞反弹

2. **第 2 关**：加入 3 个硬砖
   - 球速：1.3x (2.6, 2.6)
   - 推荐策略：优先击中硬砖两次或绕过，保持连击

3. **第 3 关**：加入最稀有的金砖
   - 球速：1.6x (3.2, 3.2)
   - 推荐策略：鉴别砖块颜色，精准定位击打顺序

4. **第 4+ 关**：无限升级
   - 球速：每关额外 +30%
   - 砖块：固定配置（40% 普通 + 40% 硬砖 + 20% 金砖）

|------|------|
| **←** | 挡板左移 |
| **→** | 挡板右移 |
| **R** | 重新开始（游戏结束时） |
| **ESC** | 退出游戏 |

## 📊 评分案例

### 场景：第 2 关的完美连击
```
连击 0-4: 击中普通砖  × 4 次 = 10×1.0 = 40 分（连击数：4）
连击 5-9: 击中硬砖第1次 × 3 次 = 25×1.2 = 90 分（连击数：7）
          击中普通砖  × 2 次 = 10×1.2 = 24 分（连击数：9）
连击 10+: 击中硬砖第2次 × 3 次 = 25×1.4 = 105 分（完成硬砖）

总分：40 + 90 + 24 + 105 = 259 分（含连击加成）
```

## 📝 代码结构优化

### Brick 类（新增）
```cpp
class Brick {
    int type;        // 1=普通, 2=硬, 3=金
    int durability;  // 剩余击数
    
    int GetPoints() const;      // 获取基础分数
    bool Hit();                 // 受击一次，返回是否被摧毁
};
```

### main.cpp（新增函数）
- `LevelData InitializeLevel(int level)`：生成关卡数据
- `void RebuildBricks(...)`：重建指定关卡的砖块阵列
- `GameState` 枚举：PLAYING / GAME_OVER / LEVEL_COMPLETE

### 游戏循环改进
- 连击计数与倍数计算
- 关卡检测与自动进度
- 命数扣减与球重置逻辑
